### Fix — "Show empty columns" had no effect for local profiles

- Cause: columns were derived only from currently-observed status values, so an emptied
  column disappeared and the toggle had nothing to filter.
- Fix: local profiles now accumulate + persist the discovered status set (`setProfileColumns`);
  `resolveColumnValues` merges observed into the known set and persists growth, so columns stay
  stable once seen and "Show empty columns" governs visibility. Starter Kit profiles use their
  fixed allowed-value columns. (Columns emptied before this fix aren't recoverable — only
  statuses seen after it are remembered.)

### Redesign — definition-driven columns (dropped observed-value columns)

- Per user feedback: columns must come from strong definitions, not from values observed in
  notes (which created stale columns from typos/invalid data).
- Removed `deriveColumns` (status.ts) and the `setProfileColumns` accumulation; deleted the
  observed-value/accumulation path in the view.

### Decisions / notes

- **Cross-lane drag is property-grouping only.** Note-type reassignment isn't safe (types are
  recognized from tags/folders), so a cross-lane drop under note-type grouping is a no-op +
  logged; intra-lane drag is unchanged. Documented as the feature's posture.
- **Mobile posture (swimlanes):** lanes stack vertically and are collapsible, which already
  acts as the accordion (one-lane-expanded) degradation; no separate mobile switcher built.
- **`obsidian plugin:reload` gotcha:** an open Bases leaf keeps its stale sub-view; must detach
    - reopen the base to force a fresh render. Captured in the plan's harness section.

### Milestone 4 — relationships, blocked_by, relational filtering (done)

- `domain/relationships.ts` (pure, tested): `resolveRelationships(records, heuristics)` combines
  **direct** role link-properties, **inverse** reverse lookup (`parent`↔`child`, `sibling`
  symmetric; `blocked_by` has no inverse), and a **link-scoped tag heuristic** (tagged note that
  links to a source ⇒ that role). Direct links may leave the board set; inverse/heuristic only
  form between known notes. `normalizeTag` helper.
- `domain/filtering.ts` (pure, tested): `passesFilter` for the blocked filter (`all`/`only`/`hide`).
- `services/relationships.service.ts`: metadata-cache bridge — tags (`getAllTags`), role links
  (`frontmatterLinks` + `getFirstLinkpathDest`, case-insensitive, array-aware), outgoing links
  (`metadataCache.resolvedLinks`). `roleProperties` resolves per-role link-property names from
  the profile (missing rule → per-role default constant; explicit empty → role disabled).
  `toCardRelationships` maps resolved paths → display-ready `{key,label}`.
- Card UI: `card-renderer.ts` renders a relationship badge row (`▲` parent / `▼` child / `↔`
  sibling + count; red `⛔` blocked badge); blocked cards get `kap-card-blocked` (red left
  accent). Badge click → `onRelationship` callback.
- View: computes `relationshipsByPath` in `rebuild`, attaches per-card relationships, applies the
  blocked filter before `buildBoard`; `blockedFilter` view option; badge/context-menu navigation
  (single related → open; multiple → cursor menu); context menu lists related notes.
- Modal: **Relationships** section — per-role link-property dropdowns + a "detect children by
  tag" heuristic text input. `setRelationships` service; default profiles seed the 4 role rules
  (`parent`/`siblings`/`children`/`blocked_by`). New constants `DEFAULT_PARENT/CHILD/SIBLING_PROPERTY`.
- CSS: relationship badge styles in `@layer kap-components` using Tailwind `@apply` for
  layout/spacing/typography and Obsidian `var(--…)` for colors (per the Tailwind hard rule).
- Tests: relationships (12) + filtering specs → 129 total. `tsc`/`lint`/`bun test`/`build` green.
- **Verified live in Obsidian:** added link-properties to fixtures (A children C/D; B blocked_by
  A; C siblings D; F blocked_by B). Resolved correctly incl. inverse (C/D parent A, C↔D siblings);
  B/F show red blocked badge + accent; clicking B's blocked badge opened Task A; Task A's children
  badge opened a C/D menu; `blockedFilter: only` → just B/F; zero console errors. Fixtures keep
  the new relationship properties as richer demo data.

### Decisions / notes (M4)

- **Heuristic is link-scoped only.** An unscoped heuristic (`requiresLinkToSource: false`) would
  match every tagged note board-wide, so it's treated as a no-op; only the tag+link form applies.
- **Relationships are read-only this milestone** — never written back (Business Rule #9: no
  auto-transition). `blocked_by` only flags + filters + navigates.
- **Role property resolution:** missing rule → default; explicit empty → disabled. Default
  profiles seed all four; the service fallback keeps pre-seed/legacy profiles working.

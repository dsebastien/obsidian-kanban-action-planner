### Milestone 1 — core board (done)

- Domain (pure, unit-tested): `domain/status.ts` (status-property detection, value
  normalization, numeric/lexical column derivation, Unmapped resolution), `domain/ordering.ts`
  (midpoint math, `planInsertion` choosing single-write vs renumber, precision-exhaustion
  guard), `domain/board-model.ts` (single-lane bucketing + sort + hide-empty-Unmapped).
- `services/frontmatter.service.ts`: cached reads, `processFrontMatter` write/delete,
  case-insensitive key matching, order coercion (pure helpers tested).
- UI (vanilla DOM, scoped): `ui/board/board-renderer.ts`, `card-renderer.ts`, and
  `dnd-controller.ts` (pointer-event drag/drop: threshold, ghost, placeholder, drop-target
  hit-testing, post-drag click suppression, reduced-motion aware).
- `views/kanban/kanban-view.ts`: assembles the board, writes status + manual order on drop
  (single midpoint write when the column is ordered, else renumber), right-click `Menu`
  (open / set status / clear status), click-to-open.
- Decision: read raw frontmatter rather than Bases `Value` objects for M1 (lossless,
  testable); a Bases-`Value` adapter will be added when formula/computed properties are
  needed. Recorded as a deviation from the plan's `value-extract` module.
- `styles.src.css`: scoped, responsive board styles (horizontal scroll, mobile scroll-snap,
  drag ghost/placeholder, focus-visible).
- `docs/`: Overview, Usage, Configuration rewritten for current behavior.
- Quality gate green: `tsc`, `lint` (0 warnings), `bun test` (79 pass), `bun run build`.
